Server Deploy & Commands

Det her er din samlede guide til backend på Hetzner VPS med Docker, Nginx og HTTPS. Du kan indsætte din rigtige IP og dit domæne i felterne herunder og gemme filen som dit faste server-dokument.

Tip: Erstat 46.62.246.141 og agent.sangmaskine.dk med dine rigtige værdier, og behold derefter denne fil som dit faste driftsdokument.

1. Log ind på serveren

ssh root@DIN_SERVER_IP

2. Opdater serveren

apt update && apt upgrade -y

3. Installér Docker og Compose

A

Docker

apt install -y docker.io
systemctl enable docker
systemctl start docker
docker --version
B

Docker Compose plugin

apt install -y docker-compose-plugin
docker compose version

4. Opret projektmappe

mkdir -p /opt/madkontrollen-agent
cd /opt/madkontrollen-agent

5. Backend struktur på serveren

/opt/madkontrollen-agent
├─ main.py
├─ requirements.txt
├─ Dockerfile
├─ docker-compose.yml
├─ .env
├─ app/
└─ data/

6. Opret .env

nano .env
APP_NAME=Madkontrollen Agent Backend
APP_VERSION=0.2.0
CONNECTOR_TOKEN=MEGET_LANGT_HEMMELIGT_TOKEN
DATABASE_URL=sqlite:///./data/agent.db
CORS_ORIGINS=https://agent.sangmaskine.dk

7. docker-compose.yml

version: "3.9"

services:
    backend:
        build: .
        container_name: madkontrollen-agent-backend
        restart: unless-stopped
        env_file:
            - .env
        volumes:
            - ./data:/app/data
        ports:
            - "127.0.0.1:8000:8000"

8. Start backend

docker compose up -d --build
docker compose logs -f

Test lokalt på serveren:

curl http://127.0.0.1:8000/health

Forventet svar: {"ok":true,"status":"healthy"}

9. Installér Nginx

apt install -y nginx

10. Nginx config

nano /etc/nginx/sites-available/madkontrollen-agent
server {
        listen 80;
        listen [::]:80;
        server_name agent.sangmaskine.dk;

        client_max_body_size 25m;

        location / {
                proxy_pass http://127.0.0.1:8000;
                proxy_http_version 1.1;

                proxy_set_header Host $host;
                proxy_set_header X-Real-IP $remote_addr;
                proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
                proxy_set_header X-Forwarded-Proto $scheme;

                proxy_read_timeout 300;
                proxy_connect_timeout 60;
                proxy_send_timeout 300;
        }
}
ln -s /etc/nginx/sites-available/madkontrollen-agent /etc/nginx/sites-enabled/madkontrollen-agent
nginx -t
systemctl reload nginx

11. HTTPS med Certbot

apt install -y certbot python3-certbot-nginx
certbot --nginx -d agent.sangmaskine.dk

Test certifikat-fornyelse:

certbot renew --dry-run

12. DNS hos Hetzner

Opret en A-record for dit subdomæne:

Host: agent
Type: A
Value: DIN_SERVER_IP

13. Connector config på din PC

Filen ligger typisk i config/connector.json.

{
    "connector_id": "madkontrollen-pc-1",
    "device_name": "Michael-PC",
    "base_url": "https://agent.sangmaskine.dk",
    "token": "MEGET_LANGT_HEMMELIGT_TOKEN",
    "poll_interval_seconds": 5,
    "request_timeout_seconds": 30,
    "workspace_root": "D:\\madkontrol-app",
    "log_dir": "data\\logs",
    "backup_dir": "data\\backups"
