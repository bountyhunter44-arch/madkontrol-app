Hvordan flowet skal føles

Hele pointen er, at medarbejderen kun ser det nødvendige og hurtigt kan komme i gang.

1

Ansvarlig starter dagen

Køkkenchef eller ansvarlig bruger trykker på Start dag for den aktuelle lokation.

2

Dagens opgaver genereres

Temperatur, rengøring, modtagekontrol og lukkerutiner oprettes automatisk.

3

Medarbejder ser kun det relevante

Ingen unødigt rod. Kun dagens handlinger og tydelige knapper til udførsel.

4

Alt gemmes til historik

Senere kan rapporter, afvigelser og kontrolbesøg læse direkte fra dagens registreringer.

Fokus på mobil først

Når dagen er startet, skal mobilen være arbejdsfladen. Det skal være hurtigt, simpelt og uden støj.

Det vigtige her

Når Start dag er trykket, skal medarbejderen ikke mødes af 20 links. Kun dagens opgaver, status og tydelige handlinger.

Vis kun dagens opgaver Skjul støj, sekundære moduler og administration på mobil.
Små hurtige handlinger Gem, afvigelse, ikke relevant og temperaturregistrering skal være lette.
Tydelig driftstatus Er stedet lukket eller i ferie, må systemet ikke vise falske mangler.
Klar til database UI er nu koblet til callable backend til oprettelse af dagens opgaver.
Start dag-siden er nu koblet til backend og kan oprette dagens task instances fra task templates.
