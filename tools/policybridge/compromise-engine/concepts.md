# Compromise Engine Concepts

## Purpose

The compromise engine should find possible bridges between positions without erasing real disagreement.

## Core Features

- Overlap detection.
- Conflict mapping.
- Shared value extraction.
- Constraint mapping.
- Tradeoff explanation.
- Compromise candidate generation.
- Feasibility scoring with uncertainty.

## Overlap Detection

Detect overlap in:

- Goals.
- Target groups.
- Time horizons.
- Funding principles.
- Implementation mechanisms.
- Risk tolerance.
- Values.

## Conflict Mapping

Map conflicts in:

- Ideology.
- Funding.
- Distributional impact.
- Timeline.
- Legal constraints.
- Administrative capacity.
- Stakeholder burden.

## Compromise Candidate

A future candidate should include:

- Proposal text.
- Who might support it.
- Who might object.
- What it preserves.
- What it sacrifices.
- Implementation risks.
- Financing assumptions.
- Confidence label.

## Anti-Pattern

Do not imply that compromise is always better than principled disagreement.
