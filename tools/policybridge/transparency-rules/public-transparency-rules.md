# Public Transparency Rules

## Required Public Disclosures

Every public-facing PolicyBridge analysis should explain:

- What input was analyzed.
- Who supplied it.
- What AI role generated the analysis.
- Which claims are facts.
- Which claims are estimates.
- Which claims are value judgments.
- What is uncertain.
- Which groups are affected.
- What alternatives exist.
- Whether a human reviewed the output.

## Source Labels

Use these labels:

- `source_provided`: evidence supplied in the input.
- `source_verified`: evidence checked by an approved future process.
- `participant_claim`: claim made by a stakeholder.
- `model_inference`: AI inference from available context.
- `expert_review_needed`: claim requires domain expert review.
- `time_sensitive`: claim may change quickly.

## Public Correction Rule

Future public outputs should support corrections that show:

- What changed.
- Why it changed.
- Who reviewed it.
- Whether conclusions changed.
