# PolicyBridge Foundation

PolicyBridge is a planning package for an AI-based policy and governance simulator. This folder contains only documentation, registries, governance boundaries, workflow definitions, AI role descriptions, and public explanation templates.

There is no runtime implementation in this phase.

## Purpose

PolicyBridge should help citizens, politicians, journalists, organizations, and advisors understand policy consequences, compromise options, financing realism, implementation risks, party overlap, long-term effects, and realistic scenarios.

AI is allowed to act as:

- Analysis tool.
- Compromise finder.
- Consequence simulator.
- Transparency engine.
- Policy sparring partner.

AI must not act as:

- Absolute truth.
- Hidden partisan persuasion.
- Fact invention engine.
- Voter manipulation system.
- Oracle that hides uncertainty.

## Required Directories

- `registries/`: policy, stakeholder, module, workflow, dependency, and boundary registries.
- `workflows/`: docs-only workflows for future analysis and review.
- `ai-profiles/`: role definitions and behavioral limits.
- `governance/`: neutrality, ownership, safety, and implementation gates.
- `moderation/`: non-propaganda and abuse-prevention principles.
- `public-explanations/`: templates for explaining analysis to the public.
- `compromise-engine/`: compromise and overlap concepts.
- `scenario-simulation/`: consequence and feasibility concepts.
- `transparency-rules/`: public trust, uncertainty, and disclosure rules.
- `architecture/`: maps and future module architecture.

## Current Phase

Foundation/tooling/docs only.

No backend. No realtime engine. No deploy. No auth/payment changes. No database rules. No production task engine changes.
