# Future Architecture

## Phase 0: Current Package

Docs, registries, workflows, profile definitions. No runtime.

## Phase 1: Prototype UI

Possible future scope:

- Static debate room mockup.
- AI profile selector.
- Episode structure editor.
- Summary preview using mocked data.

Requires approval before implementation.

## Phase 2: Text-Based Session Runtime

Possible future scope:

- Human text turns.
- AI text participants.
- Synthesis checkpoints.
- Moderation labels.

Requires privacy, retention, and storage design.

## Phase 3: Public Episode Pages

Possible future scope:

- Published episode summaries.
- Transparency ledger.
- Correction notes.
- Source panels.

Requires public publishing and review workflow.

## Phase 4: Realtime Voice

Possible future scope:

- Speech capture.
- Live transcription.
- Voice agent turn-taking.
- Latency management.

Not part of this task.

## Phase 5: Trust and Governance Layer

Possible future scope:

- Human moderator console.
- Audit trail.
- Appeal and correction workflows.
- Public methodology page.

## Protected Integration Gates

Before any future implementation touches auth, payment, database rules, user profiles, or deployment configuration, a separate approval and architecture review is required.
