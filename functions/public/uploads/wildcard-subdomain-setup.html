SEO Generator

Wildcard Subdomæne Setup

Opsætning af *.madkontrollen.dk til automatisk SEO-genererede kundesider. Hver kunde får deres eget subdomæne (f.eks. pizzeria-roma.madkontrollen.dk).

Server IP
46.62.246.141
Wildcard Pattern
*.madkontrollen.dk
Backend
Firebase Cloud Functions
Function
seoSiteRenderer

1. Hent Hetzner API Token

Du skal bruge et Hetzner API token til DNS-validering af wildcard SSL-certifikat.

  1. Log ind på Hetzner Cloud Console
  2. Gå til dit projekt → Security → API Tokens
  3. Klik "Generate API Token"
  4. Navn: "Certbot DNS Validation"
  5. Permissions: Read & Write
  6. Kopiér tokenet (vises kun én gang!)

2. Opsæt DNS Wildcard Record

Log ind på Hetzner DNS Console og opret en wildcard A-record:

Host
*
Type
A
Value
46.62.246.141
TTL
3600
Dette gør at ALLE subdomæner under madkontrollen.dk peger på din VPS.

3. SSH ind på serveren

ssh root@46.62.246.141

4. Installér Certbot DNS Plugin

apt update
apt install -y certbot python3-certbot-nginx python3-certbot-dns-hetzner

5. Opret Hetzner API credentials fil

mkdir -p /root/.secrets/certbot
nano /root/.secrets/certbot/hetzner.ini

Indsæt i filen (erstat med dit rigtige token):

dns_hetzner_api_token = DIT_HETZNER_API_TOKEN_HER

Sikr filen:

chmod 600 /root/.secrets/certbot/hetzner.ini

6. Anmod om Wildcard SSL Certifikat

certbot certonly \
  --dns-hetzner \
  --dns-hetzner-credentials /root/.secrets/certbot/hetzner.ini \
  -d madkontrollen.dk \
  -d *.madkontrollen.dk \
  --email din@email.dk \
  --agree-tos \
  --non-interactive
Certifikatet gemmes i: /etc/letsencrypt/live/madkontrollen.dk/

7. Opret Nginx Wildcard Konfiguration

nano /etc/nginx/sites-available/madkontrollen-wildcard

Indsæt denne konfiguration:

# HTTP → HTTPS redirect
server {
    listen 80;
    listen [::]:80;
    server_name *.madkontrollen.dk;
    
    return 301 https://$host$request_uri;
}

# HTTPS wildcard proxy til Firebase Cloud Functions
server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name *.madkontrollen.dk;

    # Wildcard SSL certifikat
    ssl_certificate /etc/letsencrypt/live/madkontrollen.dk/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/madkontrollen.dk/privkey.pem;
    
    # SSL optimering
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;

    client_max_body_size 10m;

    # Proxy til Firebase Cloud Function (seoSiteRenderer)
    location / {
        # Firebase Cloud Functions URL
        proxy_pass https://us-central1-madkontrollen.cloudfunctions.net/seoSiteRenderer;
        
        proxy_http_version 1.1;
        proxy_ssl_server_name on;
        
        # Videresend original host header
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
        
        # Timeouts
        proxy_read_timeout 300;
        proxy_connect_timeout 60;
        proxy_send_timeout 300;
        
        # Buffering
        proxy_buffering on;
        proxy_buffer_size 4k;
        proxy_buffers 8 4k;
    }

    # Health check endpoint
    location /health {
        access_log off;
        return 200 "OK\n";
        add_header Content-Type text/plain;
    }
}

8. Aktivér Nginx Konfiguration

ln -s /etc/nginx/sites-available/madkontrollen-wildcard /etc/nginx/sites-enabled/
nginx -t
systemctl reload nginx
Hvis "nginx -t" siger "syntax is ok" og "test is successful", er du klar!

9. Test Wildcard Subdomæne

Test at et vilkårligt subdomæne virker:

curl -I https://test-restaurant.madkontrollen.dk

Eller åbn i browser:

https://pizzeria-roma.madkontrollen.dk
https://restaurant-noma.madkontrollen.dk
https://cafe-europa.madkontrollen.dk

10. Auto-fornyelse af Wildcard Certifikat

Certbot sætter automatisk cron job op til fornyelse. Test det virker:

certbot renew --dry-run
Certifikatet fornyes automatisk hver 60. dag.

11. Firebase Cloud Function Deployment

Sørg for at din seoSiteRenderer Cloud Function er deployed:

cd d:\madkontrol-app
firebase deploy --only functions:seoSiteRenderer
Cloud Function URL: https://us-central1-madkontrollen.cloudfunctions.net/seoSiteRenderer

12. Sådan virker flowet

  1. Kunde besøger pizzeria-roma.madkontrollen.dk
  2. DNS wildcard record sender til 46.62.246.141
  3. Nginx modtager request med Host header pizzeria-roma.madkontrollen.dk
  4. Nginx proxy'er til Firebase Cloud Function seoSiteRenderer
  5. Cloud Function læser Host header, slår subdomæne op i Firestore
  6. Cloud Function returnerer SEO-optimeret HTML til kunden

14. Fejlsøgning

Problem
Subdomæne virker ikke
Tjek DNS er propageret: nslookup test.madkontrollen.dk
Problem
SSL fejl
Tjek certifikat: certbot certificates
Problem
502 Bad Gateway
Tjek Cloud Function er deployed og kører
Problem
Timeout
Øg proxy_read_timeout i Nginx config
